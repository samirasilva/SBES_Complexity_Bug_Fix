# An Exploratory Study on the Relationship Between Code Complexity and Bug-Fixing Activity in Open-Source Projects

**Paper Title**: An Exploratory Study on the Relationship Between Code Complexity and Bug-Fixing Activity in Open-Source Projects

**Submitted to**: SBES 2026 - Trilha de Ideias Inovadoras e Resultados Emergentes 

**Abstract**: Code complexity has long been considered a key factor influencing software quality, particularly in relation to defect proneness. However, existing empirical studies report mixed results, and often focus on large-scale or industrial systems. In this paper, we present an exploratory empirical study investigating the relationship between code complexity and bug-fixing activity in small-to-medium open-source projects. We analyze a set of GitHub repositories, where code complexity is measured using cyclomatic complexity, and bug-fixing activity is identified through commit messages containing bug-related keywords. Our study examines whether more complex files are more frequently associated with bug-fixing changes, and to what extent such a relationship can be observed across different projects. The results provide insights into the strength and consistency of the correlation between complexity and bug-fixing frequency, highlighting variations across projects and identifying potential thresholds where complexity may become more error-prone. We also discuss implications for software maintenance and quality assurance, as well as threats to validity. Our findings contribute to the ongoing discussion on the role of code complexity in software quality, offering a lightweight and replicable approach that can support further empirical investigations in diverse development contexts.
